# 柴牡开郁颗粒真实世界随访与成果转化平台升级报告

## 一、升级目标

本次升级围绕“成果转化展示、临床小范围试用准备、软件著作权材料准备和真实世界研究数据闭环”进行系统化优化。系统定位为医疗机构制剂柴牡开郁颗粒的随访管理、应用观察、安全性记录和科研数据采集工具。

系统不提供在线诊疗，不自动诊断，不自动开方，不调整用药剂量，不替代医生线下诊疗。正式用于临床前需通过医院伦理审批、信息安全审批和数据合规审查。

## 二、统一演示数据口径

当前展示页、数据统计页、PPT大屏、评审导出包和成果转化摘要统一使用以下脱敏演示口径：

| 指标 | 数值 |
| --- | --- |
| 建档患者数 | 120例 |
| 有效随访记录 | 486条 |
| 综合随访完成率 | 87.5% |
| 数据完整率 | 93.2% |
| 已记录不良事件 | 3例 |
| 医生已审核记录 | 428条 |

说明：以上均为脱敏演示数据，不代表真实临床疗效、安全性或统计学结论。

## 三、核心页面升级

1. 首页：强化成果转化定位、真实世界证据闭环和扫码随访流程。
2. 展示页：加入数据闭环、三位一体成果闭环、图表化指标和脱敏导出说明。
3. 数据统计页：重构为图表化数据统计页面，突出完成率、量表趋势、安全性记录、患者画像、数据质量和单患者轨迹。
4. 医生工作台：强化快捷入口、风险提示、二维码随访链接、医生综合观察评价和不良事件处理记录。
5. 患者随访页：保留温和合规提示、知情同意、当前随访节点、量表折叠填写和安全性记录。
6. 文档说明页：展示软著、研究方案、数据安全、风险预警和部署材料。

## 五、合规与安全优化

1. 不良事件口径统一为“已记录不良事件”和“不良事件记录情况”。
2. 医生评价统一为“医生综合观察评价”。
3. 睡眠量表展示统一为“睡眠自评简表”，并说明其用于随访观察，不等同于诊断工具。
4. 数据统计页默认脱敏，展示 research_id，不展示姓名和手机号。
5. 医生端可查看本人管理患者，并对手机号进行脱敏显示。
6. 审计日志覆盖患者建档、二维码生成、随访提交、医生评价、不良事件处理、数据导出等关键操作。
