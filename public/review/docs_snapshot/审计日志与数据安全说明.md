# 审计日志与数据安全说明

## 审计日志记录范围

系统记录以下关键操作：

- 创建患者
- 修改患者信息
- 提交随访
- 医生备注
- 医生综合观察评价
- 不良事件处理
- token 禁用或启用
- 数据导出

## 审计日志字段

审计日志包含操作人、角色、操作类型、操作对象、操作时间和操作内容摘要。患者通过随访链接提交时，操作人记录为患者对应研究编号或 patient_id。

## 权限控制

医生只能查看本人管理患者；管理员可查看全部患者和审计日志；患者只能通过随机 token 访问自己的随访页面。患者 token 默认长期有效，医生可在患者详情页禁用或重新启用。

## 隐私控制

系统不采集身份证号。手机号为可选字段，医生端和管理员端默认显示脱敏手机号。数据统计页默认以 research_id 作为主要识别字段。数据导出前提示脱敏要求。

## 数据保存

数据库默认本地保存于 `data/chaimu_followup.sqlite`。部署到医院内网或云服务器时，应结合服务器权限、备份策略、访问控制和日志留存要求进行配置。

## 合规要求

正式用于临床前需通过医院伦理审批、信息安全审批和数据合规审查。
