# 软件著作权说明书

## 软件名称

柴牡开郁颗粒真实世界研究与患者随访管理系统

## 软件定位

本软件面向医疗机构制剂柴牡开郁颗粒的真实世界随访管理、应用观察、安全性记录和科研数据采集。系统围绕“患者建档、随访链接生成、知情同意确认、患者随访填写、医生趋势查看、风险预警、数据统计、脱敏导出、研究摘要生成”形成数据闭环。

## 主要用户

- 管理员：负责医生账号、全部患者数据、数据统计、数据导出和审计日志查看。
- 医生：负责本人管理患者建档、随访链接生成、趋势查看、风险处理、备注记录和医生综合观察评价。
- 患者：通过随机 token 随访链接或二维码进入，仅填写自己的随访表。

## 主要功能

1. 患者档案管理：自动生成 patient_id 与 research_id，记录姓名、性别、年龄、中医诊断、证型、开始用药日期、管理医生、知情同意状态和备注。
2. 随访计划管理：自动生成第0天基线、第7天、第14天、第28天、第56天、第84天计划，并支持医生自定义随访日期。
3. 患者随访采集：采集服药情况、睡眠、情绪、主要症状、安全性、不良事件记录、患者备注。
4. 量表模块：支持睡眠自评简表、GAD-7、PHQ-9，系统计算分值并提示医生参考。
5. 服药依从性评分：根据忘记服药、自行停药、因不适停药、规律服用自动计算 adherence_score 与分级。
6. 医生工作台：患者列表、患者详情、随访完成情况、趋势图、风险预警、医生备注、不良事件处理和医生综合观察评价。
7. 数据统计：完成率、基线对比、84天趋势、量表趋势、依从性分布、医生综合观察评价分布、安全性分布、单个患者轨迹。
8. 数据导出：支持 CSV 和 Excel，支持脱敏导出，仅保留 research_id。
9. 审计日志：记录创建患者、提交随访、医生备注、医生综合观察评价、不良事件处理、token 启用禁用和数据导出等操作。

## 技术架构

- 前端：HTML、CSS、JavaScript、Chart.js
- 后端：Node.js、Express
- 数据库：SQLite/sql.js，本地文件保存
- 导出：CSV、Excel
- 部署：支持本地电脑、医院内网服务器或云服务器运行

## 数据安全设计

系统不采集身份证号；手机号为可选字段，页面默认脱敏显示；患者访问链接使用随机 token；医生仅能查看本人管理患者，管理员可查看全部患者；导出前提示脱敏；审计日志记录关键操作。

## 合规边界

本软件仅用于患者随访、应用观察、不良事件记录和真实世界研究数据采集，不提供在线诊疗，不替代医生诊疗，不生成诊断结论，不自动开方，不提供剂量调整建议，不接入互联网医院，不包含支付功能。正式用于临床前需通过医院伦理审批、信息安全审批和数据合规审查。
