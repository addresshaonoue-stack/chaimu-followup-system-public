# 系统功能模块说明

## 1. 首页与系统展示

首页呈现系统定位、角色范围、随访周期、导出能力和项目特色。展示页用于汇总呈现数据闭环、核心指标、趋势图、患者样例数据和真实世界研究流程。

## 2. 用户登录与权限

系统包含管理员和医生登录。管理员可创建医生账号、查看全部患者、导出数据、查看统计和审计日志。医生只能查看本人管理患者。患者不需要注册，通过专属随机 token 链接或二维码进入随访页面。

## 3. 患者档案

患者建档时自动生成 patient_id、research_id 和随机 token。档案字段包括姓名、性别、年龄、手机号、诊断、证型、开始用药日期、管理医生、知情同意状态和备注。手机号页面显示时默认脱敏。

## 4. 随访计划

系统自动生成第0天基线、第7天、第14天、第28天、第56天、第84天随访计划。医生可为患者添加自定义随访节点。

## 5. 患者随访表

随访表采集服药情况、睡眠、情绪、主要症状、安全性、不良反应和备注。患者首次填写前必须确认知情同意提示，系统记录 consent_signed 与 consent_time。

## 6. 量表模块

随访表内置可折叠量表模块，降低患者填写压力。包括 PSQI 简化睡眠评估、GAD-7 焦虑量表、PHQ-9 抑郁量表。系统自动计算总分和分级，并进入医生端、数据统计页和导出数据。

## 7. 服药依从性模块

系统采集忘记服药、因症状改善自行停药、因不适自行停药、是否按医嘱规律服用四项信息，自动生成 adherence_score 和高/中/低依从性分级。

## 8. 医生工作台

医生端包含患者列表、患者详情、二维码/随访链接、随访完成情况、趋势图、量表趋势、风险预警、不良反应处理、医生备注和医生疗效评价。

## 9. 数据统计

数据统计页包含总患者数、基线人数、各节点完成率、睡眠/焦虑/抑郁趋势、PSQI/GAD-7/PHQ-9趋势、基线对比、依从性分布、疗效评价分布、安全性评价分布、不良反应列表、完成矩阵和单个患者轨迹。

## 10. 数据导出

系统支持 CSV 和 Excel 导出。导出字段包括 research_id、patient_id、性别、年龄、诊断、证型、开始用药日期、各随访节点评分、量表总分与分级、依从性评分与分级、不良反应、医生评价和医生备注。脱敏导出时不导出姓名和手机号。

## 11. 审计日志

管理员可查看审计日志。记录创建患者、修改患者信息、提交随访、医生备注、医生疗效评价、不良反应处理、token 禁用/启用和数据导出等操作。
