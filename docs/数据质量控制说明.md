# 数据质量控制说明

## 数据完整性

系统按固定 84 天随访节点生成计划，并在数据统计页显示应随访次数、已完成次数、缺失次数、失访患者数、补访记录数和数据完整率。

## 字段校验

- 0-10 分评分字段限制在 0 至 10。
- PSQI 简表、GAD-7、PHQ-9 各条目限制在 0 至 3。
- GAD-7、PHQ-9 自动计算总分和分级，减少人工计算错误。
- 依从性评分由四个是/否字段自动计算，减少手工录入偏差。
- 不良反应严重程度限制为轻、中、重。

## 知情同意

患者首次提交随访前必须确认知情同意提示，系统记录 consent_signed 和 consent_time。医生端和管理员端可查看知情同意状态。

## 数据追溯

审计日志记录关键操作，包括创建患者、提交随访、医生备注、医生疗效评价、不良反应处理、token 启用禁用和数据导出。科研数据如需复核，可结合随访记录、医生备注、处理记录和审计日志进行追溯。

## 脱敏控制

手机号页面显示默认脱敏。数据统计页和数据导出页提供脱敏导出入口，脱敏导出仅保留 research_id，不导出姓名和手机号。

## 统计核查

数据统计页提供完成矩阵、基线对比表、量表趋势图、依从性与完成率表、不良反应列表和安全性分布图，用于发现缺失节点、异常记录和数据质量问题。
