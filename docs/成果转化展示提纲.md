# 成果转化展示提纲

## 一、项目名称

柴牡开郁颗粒真实世界研究与患者随访管理系统。

## 二、一句话定位

面向医疗机构制剂用药后观察的真实世界研究数据闭环工具，服务患者随访、医生管理、数据统计、数据导出和成果材料沉淀。

## 三、展示主线

1. 临床科研痛点：用药后随访分散、数据结构不统一、疗效变化难以连续追踪、安全性记录难以沉淀。
2. 系统解决方案：患者建档后生成专属链接和二维码，患者按节点填写随访，医生查看趋势和风险，数据统计页形成统计、导出和研究摘要。
3. 数据闭环证据：系统支持第0天基线、第7天、第14天、第28天、第56天、第84天节点，记录睡眠、情绪、症状、安全性、量表和依从性。
4. 临床试用基础：医生工作台可查看患者列表、下次随访、逾期状态、风险等级、个人趋势、备注和处理记录。
5. 科研转化基础：科研页提供基线对比、完成率、数据质量、PSQI/GAD-7/PHQ-9趋势、不良反应分布、依从性分布和 CSV/Excel 导出。

## 四、展示流程建议

1. 打开 `/demo.html`，展示系统定位、闭环流程、核心指标和趋势图。
2. 打开医生工作台，展示患者列表、二维码链接、风险预警、医生疗效评价和个人详情。
3. 打开患者随访页，展示知情同意确认、基础随访表和可折叠量表模块。
4. 打开数据统计页，展示 84 天趋势、基线对比、量表趋势、分布图和单例下钻。
5. 打开导出页，展示脱敏导出、导出字段、预览表、CSV 和 Excel 下载能力。
6. 打开 docs 目录，展示软著说明、部署说明、数据库设计、研究方案和数据安全材料。

## 五、常见问题回应

| 问题 | 回答要点 |
|---|---|
| 系统是否真实可用 | 已具备登录、角色权限、建档、二维码、患者填报、医生查看、数据统计、导出流程 |
| 数据是否可追溯 | 系统记录知情同意时间、随访提交、医生备注、医生评价、不良反应处理和数据导出审计日志 |
| 科研是否能用 | 支持基线对比、完成率、数据质量、量表趋势、安全性和依从性统计 |
| 临床是否能落地 | 医生端关注患者随访进度、风险等级、下次随访和处理记录，适合院内小范围试用 |
| 成果材料是否完整 | 已准备软著说明、用户手册、管理员手册、模块说明、数据库设计、部署说明和截图清单 |

## 六、展示重点

- 用闭环流程讲清患者建档、随访填写、医生管理、数据统计和导出的完整链路。
- 用核心指标证明系统数据可采集、可统计、可导出。
- 用单个患者轨迹讲清随访管理价值。
- 用数据统计页证明数据能够沉淀为研究资料。
- 用脱敏导出和审计日志证明数据安全意识。
