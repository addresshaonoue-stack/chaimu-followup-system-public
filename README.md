# 柴牡开郁颗粒真实世界研究与患者随访管理系统

本系统用于医疗机构制剂柴牡开郁颗粒的真实世界随访、疗效观察、不良反应记录和科研数据采集。

## 功能范围

- 管理员：创建医生账号、查看全部患者、查看统计、导出数据。
- 医生：创建患者档案、生成随访二维码/链接、查看本人管理患者、查看趋势图、添加医生备注、处理不良反应记录、填写医生疗效评价。
- 患者：通过专属随访链接或二维码进入随访页，无需复杂注册，只能填写自己的随访表。
- 数据统计：统计总患者数、各节点完成人数、基线对比、数据质量、PSQI/GAD-7/PHQ-9趋势、不良反应、依从性、医生评价和平均评分变化，支持 CSV 和 Excel 导出。
- 成果转化展示：`/demo.html` 展示数据闭环、核心指标、趋势图、单例患者轨迹和虚拟患者样例数据。

## 真实世界研究闭环

系统支持完整闭环：

```text
患者建档 → 生成二维码/随访链接 → 患者填写随访 → 医生查看趋势 → 数据统计 → 导出CSV/Excel → 自动生成研究摘要
```

## 本地运行

```bash
npm install
npm run init-db
npm start
```

如需将演示库整理为包含缺失随访、补访、少量漏服和评分波动的真实世界展示数据，可执行：

```bash
npm run normalize-demo-data
```

默认访问地址：

```text
http://localhost:3000
```

初始化账号：

```text
管理员：admin / admin123
医生：doctor / doctor123
```

初始化数据包含 20 个模拟患者、随访记录、医生疗效评价示例和审计日志示例，并保留缺失随访、补访、少量漏服和评分波动；模拟数据中不良反应患者数为 0，自行停药人数为 0。

## 部署方式

### 本地电脑

1. 安装 Node.js 18 或更高版本。
2. 在项目目录执行 `npm install`。
3. 执行 `npm run init-db` 初始化数据库。
4. 执行 `npm start` 启动服务。
5. 浏览器访问 `http://localhost:3000`。

### 医院内网服务器

1. 将项目复制到内网服务器。
2. 设置环境变量：

```bash
PORT=3000
SESSION_SECRET=请替换为足够复杂的随机字符串
DB_PATH=/data/chaimu_followup.sqlite
```

3. 执行 `npm install --production`。
4. 首次部署执行 `npm run init-db`。
5. 使用内网反向代理或进程管理工具运行 `npm start`。
6. 由医院信息部门配置访问控制、备份、日志留存和服务器安全策略。

### 云服务器

1. 按内网服务器方式部署。
2. 必须启用 HTTPS。
3. 必须限制后台访问范围。
4. 必须建立数据库备份、权限审计和安全更新机制。

## 数据安全设计

- 不采集身份证号。
- 手机号为可选字段。
- 手机号页面显示时脱敏。
- 患者随访链接使用随机 token。
- 患者 token 默认长期有效，医生可在患者详情页禁用或重新启用。
- 患者首次填写随访前必须确认知情同意提示，系统记录 `consent_signed` 和 `consent_time`。
- 数据库默认保存在本地 `data/chaimu_followup.sqlite`。
- 数据统计页默认优先显示 `research_id`。
- 导出页会提示确认脱敏和授权要求，脱敏导出仅保留 `research_id`，不导出姓名和手机号。
- 关键操作写入审计日志，包括创建患者、提交随访、医生备注、医生疗效评价、不良反应处理、token 启用禁用和数据导出。
- CSV 导出对公式注入风险字符做基础转义。

## 随访计划

创建患者档案后自动生成以下节点：

- 第0天基线
- 第7天
- 第14天
- 第28天
- 第56天
- 第84天

医生可在患者详情页添加自定义随访日期。

## 风险预警规则

- 连续 2 次评分升高。
- 连续 7 天未随访。
- 出现中重度不良反应。
- 记录自行停药。
- 多次漏服或未按时服药。

预警仅用于提示医生查看随访信息，不构成诊断或处置建议。

## 量表与依从性

患者随访表内置可折叠量表模块：

- PSQI 简化睡眠评估：5 个条目，每项 0-3 分，自动计算 `psqi_simple_score`。
- GAD-7：7 个条目，每项 0-3 分，自动计算 `gad7_score` 和正常或轻微、轻度、中度、重度分级。
- PHQ-9：9 个条目，每项 0-3 分，自动计算 `phq9_score` 和正常或轻微、轻度、中度、中重度、重度分级。
- 服药依从性：根据忘记服药、自行停药、因不适停药、规律服用自动计算 `adherence_score` 和高/中/低依从性分级。

## 导出字段

导出字段包括：

- `research_id`
- `patient_id`（非脱敏导出）
- 性别、年龄、诊断、证型、开始用药日期
- 各随访节点睡眠、焦虑、抑郁、心烦评分
- PSQI 简表总分、GAD-7 总分和分级、PHQ-9 总分和分级
- 依从性评分和分级
- 服药情况
- 不良反应
- 医生疗效评价、医生安全性评价、医生备注和处理记录

数据统计页另支持导出统计摘要、基线对比、完成矩阵、不良反应列表、依从性列表、疗效评价分布和安全性评价分布，包含基线人数、第7/14/28/56/84天完成率、睡眠/焦虑/抑郁评分基线对比、PSQI/GAD-7/PHQ-9趋势、不良反应发生率、自行停药人数、漏服人数、数据质量指标和自动生成研究摘要。

## 生产使用建议

- 首次登录后立即修改默认密码。
- 设置强随机 `SESSION_SECRET`。
- 限制数据库文件访问权限。
- 定期备份 SQLite 数据库。
- 使用 HTTPS 或医院内网可信链路。
- 不要将 `data/` 目录下的数据库文件上传到公开仓库。
- 根据医院要求接入统一身份认证、审计日志和堡垒机。

## 合规边界

本系统仅用于患者随访、疗效观察、不良反应记录和真实世界研究数据采集。不提供在线诊疗，不替代医生诊疗，不自动诊断，不自动开方，不提供剂量调整建议，不接入互联网医院，不包含支付功能。正式用于临床前需通过医院伦理审批、信息安全审批和数据合规审查。
